    At first glance it isn’t obvious what is tricky about tricky-1.csv; however, if you try to use read_csv() you’ll hit the following problem:

    tricky1 <- read_csv("https://stat220-w25.github.io/data/tricky-1.csv")

    Why are there state abbreviations in the latitude column?!? (Because not all the rows put quotes around the “City, State” so it added an extra column!)

    One solution is to parse the file in pieces. First, we can import the first 7 rows of the file:

    tricky2_part1 <- read_csv(
  "https://stat220-w25.github.io/data/tricky-2.csv",
  n_max = 7
)

    Next, we parse the other rows, and manually specify the column names, since we are skipping the first row:

    tricky2_part2 <- read_csv(
  "https://stat220-w25.github.io/data/tricky-2.csv",
  skip = 8, 
  col_names = c("iata", "airport", "city", "state",  "latitude", "longitude")
)

    Now, we have to use some data wrangling skills to combine everything. First, we must create a state column in tricky2_part1

    tricky2_part1 <- tricky2_part1 %>%
  separate(city, into = c("city", "state"), sep = ",")

    Then, we need to bind the two sets of rows together using bind_rows():

    tricky2 <- bind_rows(tricky2_part1, tricky2_part2)
